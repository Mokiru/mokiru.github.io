## System.currentTimeMills()

该方法可以获取当前时间与1970年1月1日00:00之间的差值，单位毫秒。值的精确度取决于底层操作系统。

## 问题描述

在多线程环境下，随着线程数增加，System.currentTimeMills()的性能急速下降。

## 总结

JDK 8 Windows11 ultra9 275hx

在单线程环境下，使用System.currentTimeMills的耗时：

```java
static void test_milli_speed ()
{
    long sum = 0;
    int N = 100000000;
    long t1 = System.currentTimeMillis ();
    for (int i = 0; i < N; i++) 
        sum += System.currentTimeMillis ();
    long t2 = System.currentTimeMillis ();
    System.out.println ("Sum = " + sum + "; time = " + (t2 - t1) +
        "; or " + (t2 - t1) * 1.0E6 / N + " ns / iter");
}
```

```txt
output:
Sum = -7120136093332470407; time = 243; or 2.43 ns / iter
```

可以看到，这个结果表示该获取时间戳的方法性能很好，让我们可以将其用在大多数需要计时的接口中，例如统计http请求耗时、缓存定时清除、记录真实事件时间，因为这个方法耗时几乎可以忽略掉，并不影响实际的统计。

在hotspot中，找到该方法，这个方法依赖于操作系统：

```cpp
JVM_LEAF(jlong, JVM_CurrentTimeMillis(JNIEnv *env, jclass ignored))
  JVMWrapper("JVM_CurrentTimeMillis");
  return os::javaTimeMillis();
JVM_END
```

```cpp
jlong os::javaTimeMillis() {
  if (UseFakeTimers) {
    return fake_time++;
  } else {
    FILETIME wt;
    GetSystemTimeAsFileTime(&wt);
    return windows_to_java_time(wt);
  }
}

jlong windows_to_java_time(FILETIME wt) {
  jlong a = jlong_from(wt.dwHighDateTime, wt.dwLowDateTime);
  return (a - offset()) / 10000;
}

static jlong  _offset   = 116444736000000000;

jlong offset() {
  return _offset;
}
```

Java时间基于GetSystemTimeAsFileTime()，它返回一个FILETIME。这个结构可以追溯到32位系统，由两个32位字段dwHighDateTime和dwLowDateTime组成，它们组合在一起定义了从epoch时间开始的100纳秒间隔的64位数字。
